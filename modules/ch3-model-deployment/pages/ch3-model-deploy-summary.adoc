== Knowledge check

Test your understanding of the MaaS deployment architecture before moving on:

1. **Why must governed models be deployed using `LLMInferenceService` rather than the legacy deployment method?**
+
Legacy KServe `InferenceService` deployments bypass the MaaS gateway completely. They lack subscription-based access control, native API key authentication, and token rate limits. Only `LLMInferenceService` integrations route directly through the Kuadrant policy layer.

2. **What happens if a model is deployed and registered as a `MaaSModelRef`, but is not added to a `MaaSAuthPolicy`?**
+
The model will exist on the cluster and its API endpoint will be created, but no user will be able to route traffic to it. The gateway will reject the requests with a `403 Forbidden` error because physical network access has not been explicitly granted to any OpenShift groups.

3. **How does the Gateway API discovery endpoint (`/v1/models`) handle model visibility when a developer authenticates with an `sk-oai-` API key?**
+
The API gateway intercepts the request and validates the provided API key. Because the key is permanently bound to a specific subscription, the endpoint dynamically filters the response array to show *only* the models allowed by that exact subscription.

.🛠️ Troubleshooting: Model Deployment & Catalog (Click to expand)
[%collapsible]
====
If you encountered any issues verifying your deployments or catalog, check the following common resolutions:

**Issue: Model deployment is stuck in "Pending" or not reaching "Ready" status**

* **Check the Pod Events:** Run `oc describe pod -l serving.kserve.io/inferenceservice=llama-3-8b -n chocotech-models`. 
* **Common Causes:** This is typically caused by insufficient GPU resources on the node (the cluster cannot satisfy the `nvidia.com/gpu` request) or the storage puller pod failing to authenticate with the MinIO S3 vault.

**Issue: The model does not appear in the Gen AI Studio catalog**

* **Verify the Publisher Flag:** Ensure your `LLMInferenceService` spec included `publishToGenAIStudio: true`. 
* **Verify Registration:** Run `oc get maasmodelref -n chocotech-models`. If the object does not exist, the operator did not register the model. 
* **Check the AuthPolicy:** If you are logged into the UI but cannot see the model, ensure your OpenShift user group has been added to a `MaaSAuthPolicy` that explicitly lists the model's reference name.

**Issue: You deployed a model via the UI, but it isn't enforcing rate limits**

* **Legacy Mode Check:** You likely left the "Use legacy deployment method" box checked during the UI wizard. Delete the model and redeploy it, ensuring this box is cleared so it uses the governed architecture.

**Issue: Custom Grafana dashboards are not showing model-specific rate limiting data**

* **Telemetry Limitations:** Due to Kuadrant wasm-shim limitations, the `model` label only appears on the `authorized_hits` metric. It does not append to the `authorized_calls` or `limited_calls` HTTP 429 rejection metrics.
