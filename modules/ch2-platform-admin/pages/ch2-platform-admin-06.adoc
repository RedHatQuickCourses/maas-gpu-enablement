= 2.5: Generating API Keys

In legacy OpenShift AI environments, Platform Engineers had to create complex Kubernetes ServiceAccounts, assign RoleBindings, and extract raw JWTs via the CLI to give developers access to a model. 

OpenShift AI 3.4 eliminates this friction by introducing **Native API Key Management**. 

Developers no longer need Kubernetes CLI access. They simply log into the OpenShift AI dashboard, and the system automatically evaluates their OpenShift Group memberships (e.g., `chocotech-quality-control`) to determine their subscription priority. When they generate an API key, that key is permanently bound to their subscription limits.

== Exercise 1: Minting an API Key (The Developer Persona)

Let's switch hats. You are now a Developer on the ChocoTech Quality Control team, and you need an API key to integrate the upcoming Llama-3 model into the factory's defect-detection software.

1. Open the OpenShift AI dashboard in your web browser.
2. Ensure you are logged in with the user account that you previously added to the `chocotech-quality-control` OpenShift group.
3. In the left-hand navigation menu, expand the **Gen AI Studio** section.
4. Click on **API Keys**.
5. Click the **Create API Key** button.
6. Provide a descriptive name for your application (e.g., `QC-Defect-Scanner-App`).
7. Click **Create**.

[IMPORTANT]
====
**Secure your key immediately!** The platform will display the API key string only once. Copy this key and save it to a secure location (like a password manager or your local terminal's environment variables). You will not be able to view it again after closing the window.
====

=== Understanding Key Binding
When you clicked "Create," the MaaS controller silently queried your user identity. Because you belong to the `chocotech-quality-control` group, the MaaS controller permanently bound this specific API key to the `gold-plan` Subscription (100 priority, 50,000 tokens/min). 

If a user belongs to multiple groups (e.g., the CIO is in both Silver and Gold), the API key automatically inherits the limits of the Subscription with the highest priority number.
