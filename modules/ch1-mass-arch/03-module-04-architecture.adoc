**Issue**: Kuadrant pods not starting after creating Kuadrant CR

**Solution**:

. Check Connectivity Link operator logs: `oc logs -n openshift-operators deployment/connectivity-link-operator --tail=50`
. Verify Kuadrant CRDs are installed: `oc get crd | grep kuadrant`

**Issue**: Gateway status shows PROGRAMMED=Unknown (persists beyond 2 minutes)

**Solution**:

. Wait 60-90 seconds—the Gateway controller needs time to process the resource and provision listeners
. If still Unknown after 2 minutes, check Service Mesh control plane: `oc get pods -n openshift-ingress`
. Verify the Gateway controller is running: `oc get pods -n openshift-ingress-operator`
. Check Gateway events for clues: `oc describe gateway maas-default-gateway -n openshift-ingress`
. Common cause: Istio control plane not fully ready or Gateway API CRDs not installed

**Issue**: Gateway status shows PROGRAMMED=False

**Solution**:

. Verify Service Mesh control plane is healthy: `oc get pods -n openshift-ingress`
. Check for error messages in Gateway status: `oc describe gateway maas-default-gateway -n openshift-ingress`
. Common cause: Service Mesh operator not installed or Istio control plane not deployed
. Ensure the `openshift-ingress` namespace exists and contains running `istiod-*` and `istio-ingressgateway-*` pods

**Issue**: MaaS API pods not appearing after DataScienceCluster patch

**Solution**:

. Check DataScienceCluster status conditions: `oc get datasciencecluster default-dsc -o yaml | grep -A 10 conditions`
. Verify Kuadrant is running: `oc get pods -n openshift-operators`
. MaaS API deployment depends on Kuadrant being healthy—if Kuadrant pods are not Running, the operator will not create maas-api
. Check operator logs for specific errors: `oc logs -n redhat-ods-operator deployment/rhods-operator --tail=100`

**Issue**: Gen AI Studio not visible in dashboard after configuration patch

**Solution**:

. Verify dashboard config patch applied correctly: `oc get odhdashboardconfig odh-dashboard-config -n redhat-ods-applications -o yaml | grep -A 5 dashboardConfig`
. Clear browser cache and hard-reload the page (Ctrl+Shift+R or Cmd+Shift+R)
. Check dashboard pod logs for errors: `oc logs -n redhat-ods-applications deployment/odh-dashboard --tail=50`
. Restart the dashboard pod if needed: `oc rollout restart deployment/odh-dashboard -n redhat-ods-applications`

**Issue**: LeaderWorkerSet operator missing from operator list

**Solution**:

. Install from OperatorHub: Navigate to **Ecosystem → Software Catalog** in OpenShift console
. Search for "Leader Worker Set"
. Click **Install** and select openshift-lws-operator namespace
. Wait for CSV to show Succeeded status
. LeaderWorkerSet is required for llm-d orchestration in later modules
