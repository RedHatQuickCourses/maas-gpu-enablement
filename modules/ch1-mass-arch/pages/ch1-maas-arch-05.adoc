.🛠️ Troubleshooting: Did something go wrong? (Click to expand)
[%collapsible]
====
If your verification checks failed in the previous step, locate your issue below to resolve it before proceeding to Module 2.

**Issue: MaaS API pod is in `CrashLoopBackOff`**

* **Check the Database Secret:** The `maas-api` pod strictly requires a database connection string. Check that the PostgreSQL database secret `maas-db-config` exists in the `redhat-ods-applications` namespace.
* **Verify Secret Contents:** Run `oc extract secret/maas-db-config -n redhat-ods-applications --to=-` to ensure the `DB_CONNECTION_URL` is formatted correctly.

**Issue: Gen AI Studio is not visible in the dashboard**

* **Verify Llama Stack:** The UI relies on the Llama Stack Operator. Run `oc get datasciencecluster default-dsc -o jsonpath='{.spec.components.llamastackoperator.managementState}'`. It must be `Managed`.
* **Verify Dashboard Patch:** Run `oc get odhdashboardconfig odh-dashboard-config -n redhat-ods-applications -o yaml | grep -A 5 dashboardConfig` to ensure `genAiStudio` is set to `true`.
* **Hard Reload:** Clear your browser cache and hard-refresh the page (Ctrl+Shift+R or Cmd+Shift+R).

**Issue: Kuadrant pods are not starting after creating the Kuadrant CR**

* **Check Connectivity Link operator logs:** Run `oc logs -n openshift-operators deployment/connectivity-link-operator --tail=50`.
* **Verify Kuadrant CRDs are installed:** Run `oc get crd | grep kuadrant`.

**Issue: Gateway status shows `PROGRAMMED=False` or `Unknown`**

* **Verify Service Mesh:** Ensure the control plane is healthy by running `oc get pods -n openshift-ingress`.
* **Check for Errors:** Run `oc describe gateway maas-default-gateway -n openshift-ingress`.
* *Root Cause:* The Service Mesh operator is not installed, or the Istio control plane is not deployed. Ensure the `openshift-ingress` namespace exists and contains running `istiod-*` and `istio-ingressgateway-*` pods.
