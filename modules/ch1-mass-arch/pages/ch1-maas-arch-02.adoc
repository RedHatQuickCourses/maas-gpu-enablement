= 1.1: Understanding the GPU silo crisis

ChocoTech Industries has four teams independently managing GPU infrastructure. Let's examine the operational pain points this creates.

The current state of GPU allocations across ChocoTech's business units:

* **Quality Control**: 2x NVIDIA A100 GPUs running defect detection models
* **R&D**: 4x NVIDIA L40S GPUs for recipe optimization
* **Marketing**: 2x NVIDIA L4 GPUs for sentiment analysis 
* **Supply Chain**: 2x NVIDIA L40S GPUs for inventory prediction
== The Solution: What is Models as a Service (MaaS)?

Before we dive into the technical architecture, let's define exactly what we are building to solve this crisis. 

The Models-as-a-Service (MaaS) platform acts as a governance layer between users and the model serving infrastructure. This architecture enables centralized policy enforcement without modifying the underlying model serving components.

To achieve this, OpenShift AI 3.4 provides four core capabilities:

* **Subscription-based access control:** Define multiple subscriptions that grant specific groups access to models with configurable token limits.
* **Self-service API key management:** Users can create, list, and revoke their own API keys for model access through the Gen AI Studio.
* **Policy and quota management:** Enforce token limit policies to prevent resource exhaustion.
* **Usage tracking and observability:** Track consumption metrics for cost allocation, billing, and showback reporting to finance teams.

== The MaaS Impact: Problem vs. Payoff

How does this directly solve ChocoTech's "GPU Silo Crisis"? Review these core challenges and their corresponding MaaS solutions.

[cols="1a,1a",options="header",frame="all",grid="rows"]
|===
| The Pain (The Problem) 
| The Payoff (The MaaS Solution)

| **GPU Under-Utilization**
Quality Control’s $120,000 A100 GPUs sit idle 67% of the time, while R&D waits weeks for capacity. 
| **Dynamic Multi-Tenancy**
Hardware is centralized. MaaS allows dynamic accelerator slicing and multi-tenant sharing, driving up utilization and eliminating wait times.

| **Misallocated Expertise**
Data scientists spend 40% of their time debugging CUDA drivers and Kubernetes manifests. 
| **Simplified Consumption**
Developers skip the infrastructure completely. They access powerful models instantly via secure, OpenAI-compatible API endpoints using native API keys.

| **Redundant Deployments**
Four teams independently deploy the same Llama-3 8B model, wasting 128GB of storage and duplicate compute.
| **Centralized Catalog**
A single, highly optimized model deployment serves all business units simultaneously, drastically reducing storage and compute waste.

| **The Security "Black Box"**
Zero visibility into who is sending what data to which model.
| **Edge-Level Governance**
Centralized API gateways enforce strict authentication, zero-trust access, and token quotas directly at the network edge.

| **No Chargeback Visibility**
Finance cannot justify infrastructure costs because they cannot track which teams are consuming compute capacity. 
| **Granular Token Tracking**
Exact consumption metrics (prompt/completion tokens) are tracked per user and subscription, enabling precise capacity forecasting and accurate departmental chargebacks.
|===
