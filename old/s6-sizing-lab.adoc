=== Part C: Multi-Node Strategy Design (Cluster B)

**For Cluster B (2 nodes, each with 8× H100-80GB, connected via InfiniBand):**

**Question 1:** Can you use TP=16 across both nodes (8 GPUs per node)?

Reference Section 1 networking requirements table:

* Tensor Parallelism requires: __________ latency (fill in: <1-2 microseconds)
* InfiniBand HDR provides: ~1-5 microseconds latency
* Is this sufficient for TP? ✅ / ❌

**Answer:** ❌ TP cannot efficiently span nodes connected by InfiniBand (even though latency seems close)

**Why:** TP performs all-reduce after every layer (80+ times per forward pass). Even 2-5 μs latency per sync × 80 layers = 160-400 μs overhead. With NVLink (<1 μs per sync), overhead is only 80 μs. This 2-5× latency increase causes significant throughput degradation.

**Alternative: Hybrid TP + PP Strategy**

Design deployment using:

* **TP=8 within each node** (intra-node NVLink)
* **PP=2 across nodes** (inter-node InfiniBand tolerates higher latency)

**Configuration:**

[source,yaml]
----
vllm_args:
  --model=/models/llama-405b-int8
  --tensor-parallel-size=8      # Within each node
  --pipeline-parallel-size=2    # Across 2 nodes
  --dtype=int8
  # Total GPUs: 8 × 2 = 16 GPUs
----

====
**Symptoms of TP Across InfiniBand:**

**DCGM Metrics:**

* `DCGM_FI_DEV_GPU_UTIL` drops to **<20%** (GPUs idle 80% of time waiting for sync)
* `DCGM_FI_DEV_POWER_USAGE` remains high (~300W per GPU) despite low utilization (waiting consumes power)
* `DCGM_FI_DEV_FB_USED` shows expected ~25 GB per GPU (model loaded correctly)

**Performance Impact:**

* **Throughput collapses to ~400 tokens/sec** (vs 4,000 tokens/sec expected)
* **Effective performance: 10% of single-node TP=8** (90% loss due to network bottleneck)
* Token generation appears "stuttering" - periodic fast bursts followed by long pauses

**Root Cause Deep Dive:**

TP performs all-reduce synchronization after every transformer layer. For 80-layer model:

* **InfiniBand latency:** ~2-5 microseconds per sync operation
* **Total sync overhead:** 80 layers × 3 μs (average) = 240 μs per forward pass
* **Actual compute time per layer:** ~15 μs
* **Total compute time:** 80 layers × 15 μs = 1,200 μs per forward pass

**Time breakdown:**

* Sync overhead: 240 μs (17% of total time)
* **But:** Synchronization is blocking - all GPUs must wait for slowest sync
* In practice, sync overhead dominates due to:
** Network congestion (all 16 GPUs synchronizing simultaneously)
** InfiniBand latency variance (occasional 10-20 μs spikes)
** Operating system scheduling delays

**Result:** GPUs spend 80-90% of time waiting for network, only 10-20% computing.

**Correct Fix:**

Use hybrid TP=8 (within nodes) + PP=2 (across nodes) as designed in Part C. This configuration:

* Keeps TP synchronization on NVLink fabric (<1 μs latency)
* Uses PP for inter-node communication (1× per pipeline stage, not 80× per forward pass)
* Achieves 7,000 tokens/sec throughput (17.5× better than misconfigured TP=16)
