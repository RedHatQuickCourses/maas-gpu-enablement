= Operator Foundations and GPU Stack Deployment

Goal::

Master the foundational operator pattern that powers enterprise AI platform automation, understand the GPU software stack architecture, and deploy the complete NVIDIA GPU Operator stack using OpenShift's Operator Lifecycle Manager.

This chapter establishes the patterns required for managing GPU infrastructure at enterprise scale. You will learn why operators are the foundational building block for AI platform lifecycle management, how OpenShift's Operator Lifecycle Manager orchestrates software delivery, and gain hands-on experience deploying, troubleshooting, and managing the complete NVIDIA GPU Operator stack.

== Why This Chapter Matters

For platform engineers understanding operators isn't optional - it's foundational. Operators transform GPU infrastructure from manually-managed silos requiring 150+ hours of setup time into declarative, self-healing systems deployed in hours. This chapter teaches you the automation patterns that make GPU investments generate maximum ROI.

== Learning Objectives

On completing this chapter, you should be able to:

* **Understand the operator pattern** and explain why it's critical for enterprise AI platform automation at scale
* **Navigate OpenShift's Operator Lifecycle Manager (OLM)** architecture including Subscriptions, CSVs, InstallPlans, and CatalogSources
* **Differentiate between cluster operators and add-on operators** and make informed decisions about which operator catalogs to use for production
* **Deploy Node Feature Discovery and NVIDIA GPU Operator** using OLM with proper configuration
* **Manage operator lifecycle** including configuration changes, upgrades, rollbacks, and health monitoring
* **Troubleshoot common operator deployment issues** and perform complete operator cleanup and reinstallation

== Chapter Structure

This chapter follows a deliberate progression from **WHY** → **WHAT** → **HOW** → **DO** → **MANAGE** → **TROUBLESHOOT**:

**Section 1: The Operator Pattern for AI Platform Automation (12 minutes)**

Learn why operators exist, how they solve the GPU Silo Crisis, and the business value they deliver for MaaS deployments. Understand the evolution from manual scripts to declarative automation and the concrete ROI operators provide at scale.

**Section 2: Deconstructing the GPU Software Stack (15 minutes)**

Examine the six core components the NVIDIA GPU Operator manages: Node Feature Discovery, NVIDIA Driver Container, Container Toolkit, GPU Feature Discovery, DCGM, and DCGM Exporter. Understand each component's role and how they work together.

**Section 3: Operator Lifecycle Manager and Custom Resources (18 minutes)**

Deep dive into OpenShift's OLM architecture. Learn how Subscriptions, CSVs, and InstallPlans work together. Understand Custom Resource Definitions, Custom Resources, and Controllers. Master the difference between cluster operators and add-on operators, and understand when to use Red Hat vs. Certified vs. Community operator catalogs.

**Section 4: Lab - Deploying the GPU Operator Stack (30 minutes)**

Hands-on deployment of Node Feature Discovery and NVIDIA GPU Operator using OLM. Install operators, create Custom Resources, and verify the complete stack. This lab includes detailed explanations of what's happening behind the scenes at each step, reinforcing your understanding of the OLM workflow.

**Section 5: Operator Lifecycle Management (12 minutes)**

Learn how to manage operators after deployment. Modify configuration by editing Custom Resources, manage operator upgrades through OLM, monitor operator health, and understand rollback considerations for production AI platforms.

**Section 6: Lab - Troubleshooting and Cleanup (25 minutes)**

Practice diagnosing common operator issues through hands-on scenarios. Intentionally break operator deployments and learn how to fix them. Perform complete operator removal following best practices, then reinstall to reinforce the OLM workflow.

**Total Chapter Time:** 112 minutes (1 hour 52 minutes)

== Prerequisites

Before starting this chapter, you should:

* Have cluster administrator access to an OpenShift cluster (version 4.14 or higher)
* Have at least one worker node with NVIDIA GPU hardware
* Be familiar with Kubernetes basics (Pods, DaemonSets, Services, Namespaces)
* Have the `oc` CLI tool installed and configured
* Understand basic YAML syntax and Kubernetes resource structure

== What You'll Build

By the end of this chapter, you will have:

* Deployed Node Feature Discovery operator from the `redhat-operators` catalog
* Deployed NVIDIA GPU Operator from the `certified-operators` catalog
* Created a ClusterPolicy Custom Resource configuring the complete GPU software stack
* Verified all GPU Operator components running (driver, toolkit, device-plugin, DCGM, GFD)
* Practiced modifying operator configuration declaratively
* Performed operator troubleshooting and complete cleanup
* Reinforced understanding of the complete OLM workflow through reinstallation

Your OpenShift cluster will be ready to schedule GPU-accelerated workloads with automated driver management, self-healing components, and comprehensive monitoring - the foundation for a production-ready Models-as-a-Service platform.
