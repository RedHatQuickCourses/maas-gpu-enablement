2.4: Authorizing Gateway Access (MaaSAuthPolicy)

In the previous section, we defined the quotas using MaaSSubscription. However, as we learned in the Two-Part Access Rule, a subscription does not actually grant network access; it only dictates the speed limit once a user is inside.

To physically open the API Gateway and allow traffic to reach the models, we must deploy a MaaSAuthPolicy.

Exercise 1: Deploy the Authorization Policies

Just like the subscriptions, authorization policies must be deployed into the centralized models-as-a-service namespace.

We will create three policies to map our ChocoTech OpenShift groups (the subjects) to our target model (llama-3-8b-ref).

Execute the following block to deploy the Auth Policies:

cat <<EOF | oc apply -f -
---
apiVersion: maas.opendatahub.io/v1alpha1
kind: MaaSAuthPolicy
metadata:
  name: gold-access
  namespace: models-as-a-service
spec:
  modelRefs:
    - name: llama-3-8b-ref
      namespace: chocotech-models
  subjects:
    groups:
      - name: chocotech-quality-control
    users: []
---
apiVersion: maas.opendatahub.io/v1alpha1
kind: MaaSAuthPolicy
metadata:
  name: silver-access
  namespace: models-as-a-service
spec:
  modelRefs:
    - name: llama-3-8b-ref
      namespace: chocotech-models
  subjects:
    groups:
      - name: chocotech-rd
    users: []
---
apiVersion: maas.opendatahub.io/v1alpha1
kind: MaaSAuthPolicy
metadata:
  name: bronze-access
  namespace: models-as-a-service
spec:
  modelRefs:
    - name: llama-3-8b-ref
      namespace: chocotech-models
  subjects:
    groups:
      - name: chocotech-marketing
    users: []
EOF

Expected output: The terminal will confirm the creation of gold-access, silver-access, and bronze-access.

Verification

Verify that the MaaS operator successfully registered the authorization policies:

oc get maasauthpolicy -n models-as-a-service

You have now successfully locked down the platform!

  1. Only users in the QC, R&D, and Marketing groups can access the Gateway.

  2. When they do, their traffic will be strictly throttled based on their Subscription tier.

The platform infrastructure is officially governed. The last step before we can hand this over to the developers is providing them with a secure way to log in.
