5.4: OpenAI API compatibility and migration patterns

MaaS uses the OpenAI-compatible API format, making it easy to migrate existing applications from OpenAI to self-hosted models.

Applications built for ChatGPT or GPT-4 can switch to MaaS models by changing two parameters: base URL and API key.

OpenAI SDK compatibility

Install the OpenAI Python SDK:

pip install openai

Create a migration example showing how to switch from OpenAI to MaaS:

cat > openai_to_maas_migration.py <<'EOF'
#!/usr/bin/env python3
import os
from openai import OpenAI

# BEFORE: Using OpenAI ChatGPT
# client = OpenAI(
#     api_key=os.getenv("OPENAI_API_KEY")
# )

# AFTER: Using MaaS (only two changes required)
client = OpenAI(
    base_url=f"https://{os.getenv('MAAS_URL')}/v1",  # Change 1: MaaS gateway URL
    api_key=os.getenv("MAAS_TOKEN")                   # Change 2: MaaS Bearer token
)

# The rest of the code remains identical
response = client.chat.completions.create(
    model="meta-llama/Meta-Llama-3-8B-Instruct",  # Use MaaS model instead of gpt-4
    messages=[
        {"role": "system", "content": "You are a helpful assistant for ChocoTech Industries."},
        {"role": "user", "content": "How can we reduce production waste in chocolate manufacturing?"}
    ],
    max_tokens=200,
    temperature=0.7
)

print("Response from MaaS model:")
print(response.choices[0].message.content)
print(f"\nTokens used: {response.usage.total_tokens}")
EOF

chmod +x openai_to_maas_migration.py

Run the migration example:

python3 openai_to_maas_migration.py

Expected output:

Response from MaaS model:

Tokens used: 148

Migration checklist for existing applications

For applications using OpenAI SDK:

  1. Change base_url to MaaS gateway endpoint

  2. Replace OpenAI API key with MaaS Bearer token

  3. Update model names (gpt-4 → meta-llama/Meta-Llama-3-8B-Instruct)

  4. Test error handling for tier-specific rate limits

  5. No other code changes required

For applications using direct REST API calls:

  1. Replace https://api.openai.com/v1 with https://<maas-gateway>/v1

  2. Replace Authorization: Bearer sk-…​ with MaaS Bearer token

  3. Update model identifiers in request payloads

  4. API request/response format remains identical

Verification

Confirm OpenAI compatibility:

  • OpenAI Python SDK works with MaaS without code changes (except base_url and api_key)

  • Existing applications can migrate by changing two configuration parameters

  • Request/response format matches OpenAI API exactly

  • Same SDK patterns work for both OpenAI and MaaS
