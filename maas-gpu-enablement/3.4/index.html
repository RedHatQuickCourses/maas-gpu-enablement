Models as a Service: Breaking the GPU Silos on OpenShift AI 3.3

Welcome to this Red Hat OpenShift AI 3 - Models as a Service QuickCourse!

Course Overview: The MaaS Mandate

Designed for Platform Engineers and Developers, this course provides the exact steps to build a governed Models as a Service (MaaS) ecosystem on OpenShift AI 3.3. We skip the trial-and-error of figuring out which operators and registries to install. Instead - you will learn how to immediately transform fragmented, isolated hardware into a fairly scheduled, multi-tenant shared service, empowering your business units to deploy agentic AI smoothly, securely, and at scale.

By the end of this course, you will be equipped to empower your lines of business with self-service models to build agentic AI applications smoothly and securely.

Experience Level: Intermediate

Prerequisites

To get the most out of this QuickCourse, you should come to the table with:

  • Red Hat OpenShift administration experience (version 4.20 or later)

  • Basic Kubernetes operator management knowledge

  • Familiarity with core GPU concepts and generic GPU enablement

  • An understanding of API concepts and REST principles

  • Basic command-line experience

What You’ll Learn

In this hands-on QuickCourse, you will:

  • Master MaaS: Understand how Models as a Service solves complex GPU resource allocation challenges.

  • Architect a Hub: Design and implement a centralized model hub featuring governed, token-based consumption.

  • Configure and Deploy: Set up MaaS on OpenShift AI using operators and CRDs, and deploy generative AI models (using llm-d, vLLM, and KServe) for distributed inference.

  • Publish for Self-Service: Publish models to the OpenShift AI ModelsAsService catalog, complete with role-based access permissions.

  • Enforce Governance: Implement tiered service limits and rate limiting using Kuadrant.

  • Monitor and Report: Generate chargeback reports and track GPU utilization metrics using Grafana.

  • Integrate Seamlessly: Connect AI models into applications using OpenAI-compatible APIs with robust retry logic.

QuickCourse Environment

This lab requires a robust setup. The recommended environment is a Single Node OpenShift (SNO) cluster with multiple NVIDIA GPUs (minimum 48GB total, e.g., 2x24GB).

Pre-flight Checklist The following components must be pre-configured before you begin:

  • OpenShift Cluster: SNO with GPU nodes configured (version 4.20 or later).

  • GPU Operators: NVIDIA GPU Operator and Node Feature Discovery installed.

  • OpenShift AI Operator: Installed via the stable-3.x channel.

  • Service Mesh Operator: Installed (required for Gateway API routing).

  • Red Hat Connectivity Link: Kuadrant installed for auth and rate limiting.

  • LeaderWorkerSet Operator: Installed for llm-d (required for Mixture of Expoert (MOE) Model) inference.

  • CLI Access: Web Terminal Operator or Developer Spaces active, with the oc command accessible.

  • Serving & Drivers: KServe components deployed, and NVIDIA drivers loaded/tested.

Why do we need multiple GPUs for this lab?

  • The LlamaStack model powers the MaaS Dashboard and consumes 1 dedicated GPU.

  • It allows us to simulate realistic, multi-tenant model serving scenarios.

  • It enables proper demonstrations of load balancing and autoscaling.

  • It provides the capacity to deploy the NVIDIA-Nemotron-3-Nano-4B-FP8 (or 30B A3B) model.

Estimated Time & Agenda

  1. MaaS Architecture (30 mins)

  2. MaaS Configuration (40 mins)
