Lab: Visualizing GPU Hardware

Estimated reading time: 20 minutes.

Objective

Import and customize NVIDIA DCGM Exporter dashboards to visualize interconnected metrics, mapping standard node resources (CPU, RAM) alongside accelerator metrics including GPU power usage, temperature, tensor core utilization, and MIG instance allocations.

Before you begin

This lab requires:

  • Completed Grafana installation and datasource configuration

  • GPU metrics actively being collected by Prometheus

  • Access to the Grafana web interface

Instructions

1. Import NVIDIA DCGM Exporter Dashboard

NVIDIA provides official Grafana dashboards designed for DCGM metrics visualization.

  1. Download the official NVIDIA DCGM dashboard JSON

    $ curl -O https://raw.githubusercontent.com/NVIDIA/dcgm-exporter/main/grafana/dcgm-exporter-dashboard.json
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
100 45328  100 45328    0     0   187k      0 --:--:-- --:--:-- --:--:--  187k

  2. Create a ConfigMap with the dashboard JSON

    $ oc create configmap dcgm-dashboard --from-file=dcgm-exporter-dashboard.json -n grafana
configmap/dcgm-dashboard created

  3. Create a GrafanaDashboard CR to import it

    $ cat <<EOF | oc apply -f -
apiVersion: grafana.integreatly.org/v1beta1
kind: GrafanaDashboard
metadata:
*  name: dcgm-exporter-dashboard*
*  namespace: grafana*
spec:
*  instanceSelector:*
*    matchLabels:*
*      dashboards: "grafana"*
*  configMapRef:*
*    name: dcgm-dashboard*
*    key: dcgm-exporter-dashboard.json*
EOF
grafanadashboard.grafana.integreatly.org/dcgm-exporter-dashboard created

  4. Verify the dashboard was imported

    Log in to Grafana and navigate to Dashboards → Browse. You should see "DCGM Exporter Dashboard" in the list.

2. Explore GPU Utilization Metrics

  1. Open the DCGM Exporter Dashboard in Grafana

  2. Review the key panels:

    GPU Utilization

    Shows real-time GPU compute usage percentage across all devices. Useful for identifying idle GPUs or oversubscribed instances.

    Query example:

    DCGM_FI_DEV_GPU_UTIL{job="nvidia-dcgm-exporter"}
    GPU Memory Usage

    Displays allocated GPU memory (framebuffer) usage. Critical for identifying memory-constrained workloads.

    Query example:

    DCGM_FI_DEV_FB_USED{job="nvidia-dcgm-exporter"}
    GPU Temperature

    Monitors thermal performance. High temperatures may indicate cooling issues or need for throttling.

    Query example:

    DCGM_FI_DEV_GPU_TEMP{job="nvidia-dcgm-exporter"}
    Power Usage

    Tracks power consumption in watts. Useful for cost modeling and capacity planning.

    Query example:

    DCGM_FI_DEV_POWER_USAGE{job="nvidia-dcgm-exporter"}

3. Create a Custom GPU Overview Dashboard

Build a custom dashboard that correlates GPU metrics with node-level infrastructure metrics.

  1. In Grafana, click Dashboards → New → New Dashboard

  2. Click Add visualization

  3. Select the Prometheus datasource

  4. Create the first panel: GPU Utilization by Node

    • Panel type: Time series

    • Query:

      avg by (exported_pod) (DCGM_FI_DEV_GPU_UTIL)

    • Legend: {{exported_pod}}

    • Panel title: "GPU Utilization by Node"

    • Y-axis: 0-100 (percentage)

  5. Add a second panel: GPU Memory Utilization

    • Panel type: Time series

    • Query:

      (DCGM_FI_DEV_FB_USED / (DCGM_FI_DEV_FB_USED + DCGM_FI_DEV_FB_FREE)) * 100

    • Panel title: "GPU Memory Utilization %"

    • Y-axis: 0-100

  6. Add a panel for Node CPU vs GPU Correlation

    • Panel type: Time series

    • Add two queries:

      Query A (Node CPU):

      100 - (avg by (node) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

      Query B (GPU Utilization):

      avg by (node) (DCGM_FI_DEV_GPU_UTIL)

    • Panel title: "CPU vs GPU Utilization"

    • This visualization helps identify whether GPU or CPU is the bottleneck

  7. Add a panel for GPU Power and Temperature

    • Panel type: Time series

    • Query A (Power):

      DCGM_FI_DEV_POWER_USAGE

    • Query B (Temperature):

      DCGM_FI_DEV_GPU_TEMP

    • Use dual Y-axis: left for Power (W), right for Temperature (°C)

  8. Add a stat panel for Total GPUs Available

    • Panel type: Stat

    • Query:

      count(DCGM_FI_DEV_GPU_UTIL)

    • Panel title: "Total GPU Devices"

  9. Save the dashboard with name: "GPU Infrastructure Overview"

4. Create MIG Instance Monitoring Dashboard

For clusters using Multi-Instance GPU, create a dashboard to monitor MIG allocations.

  1. Create a new dashboard: MIG Instance Utilization

  2. Add panel: MIG Instance Allocation

    • Panel type: Gauge

    • Query:

      sum by (GPU_I_ID, GPU_I_PROFILE) (DCGM_FI_PROF_GR_ENGINE_ACTIVE)

    • Group by MIG profile and instance ID

    • Panel title: "MIG Instance Activity"

  3. Add panel: MIG Memory Usage by Profile

    • Panel type: Bar gauge

    • Query:

      sum by (GPU_I_PROFILE) (DCGM_FI_DEV_FB_USED{GPU_I_PROFILE!=""})

    • Panel title: "Memory Usage by MIG Profile"

    • Shows memory consumption per MIG profile type (1g.10gb, 2g.20gb, etc.)

  4. Add panel: MIG Instances per Node

    • Panel type: Table

    • Query:

      count by (node, GPU_I_PROFILE) (DCGM_FI_DEV_FB_USED{GPU_I_PROFILE!=""})

    • Format: Table

    • Shows distribution of MIG instances across nodes

5. Set Up GPU Health Alerts

Configure alerts for critical GPU conditions.

  1. Navigate to Alerting → Alert rules → New alert rule

  2. Create alert: High GPU Temperature

    • Alert name: GPU High Temperature

    • Query:

      DCGM_FI_DEV_GPU_TEMP > 85

    • Condition: Alert if query returns value above 85°C

    • Evaluation interval: 1 minute

    • Pending period: 5 minutes

    • Summary: "GPU {{$labels.device}} temperature is {{$value}}°C"

  3. Create alert: GPU Memory Exhaustion

    • Alert name: GPU Memory Near Full

    • Query:

      (DCGM_FI_DEV_FB_USED / (DCGM_FI_DEV_FB_USED + DCGM_FI_DEV_FB_FREE)) > 0.9

    • Condition: Alert if memory usage exceeds 90%

    • Summary: "GPU {{$labels.device}} memory usage at {{$value}}%"

  4. Create alert: GPU Utilization Anomaly

    • Alert name: GPU Idle Despite Workload

    • Query:

      DCGM_FI_DEV_GPU_UTIL < 10 and on(node) kube_pod_container_resource_requests{resource="nvidia.com/gpu"} > 0

    • Condition: GPU allocated but utilization < 10%

    • Summary: "GPU allocated but underutilized on {{$labels.node}}"

6. Create an Executive Summary Dashboard

Build a high-level dashboard for management visibility.

  1. Create dashboard: GPU Fleet Overview

  2. Add panel: Total GPU Capacity

    • Panel type: Stat

    • Query:

      count(DCGM_FI_DEV_GPU_UTIL)

    • Large font, prominent display

  3. Add panel: Average GPU Utilization

    • Panel type: Gauge

    • Query:

      avg(DCGM_FI_DEV_GPU_UTIL)

    • Thresholds: Green (>70%), Yellow (40-70%), Red (<40%)

  4. Add panel: GPU Hours Consumed (24h)

    • Panel type: Stat

    • Query:

      sum(increase(DCGM_FI_DEV_GPU_UTIL[24h])) / 100

    • Shows total GPU-hours used in last 24 hours

  5. Add panel: Cost Projection

    If you have cost-per-GPU-hour defined (e.g., $0.50/hour for A100):

    • Panel type: Stat

    • Query:

      (sum(increase(DCGM_FI_DEV_GPU_UTIL[24h])) / 100) * 0.50 * 30

    • Shows estimated monthly cost at current utilization

    • Panel title: "Estimated Monthly GPU Cost"

You have successfully created comprehensive GPU observability dashboards that visualize hardware telemetry, correlate infrastructure metrics, monitor MIG instances, configure health alerts, and provide executive-level reporting. This observability foundation enables data-driven optimization of your Models-as-a-Service GPU infrastructure.
