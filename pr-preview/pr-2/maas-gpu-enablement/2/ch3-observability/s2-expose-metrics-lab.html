Lab: Exposing GPU Metrics

Estimated reading time: 20 minutes.

This lab in a work in progress. The content is being developed and may contain placeholders, incomplete information. Please refer to the final version for the complete lab experience.
Objective

Enable OpenShift user-workload monitoring by modifying the cluster-monitoring-config ConfigMap, and label the GPU Operator namespace to expose GPU telemetry for Prometheus scraping via the NVIDIA DCGM Exporter.

Before you begin

This lab requires:

  • Completed GPU Operator installation with DCGM Exporter enabled

  • Cluster administrator privileges

  • Basic understanding of Prometheus and OpenShift monitoring architecture

Instructions

1. Enable User Workload Monitoring

OpenShift separates platform monitoring from user workload monitoring. GPU metrics are considered user workload metrics and require this feature to be enabled.

  1. Check current cluster monitoring configuration

    $ oc get configmap cluster-monitoring-config -n openshift-monitoring -o yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: cluster-monitoring-config
  namespace: openshift-monitoring
data:
  config.yaml: |
    # ... existing configuration

  2. Enable user workload monitoring

    $ cat <<EOF | oc apply -f -
apiVersion: v1
kind: ConfigMap
metadata:
*  name: cluster-monitoring-config*
*  namespace: openshift-monitoring*
data:
*  config.yaml: |*
*    enableUserWorkload: true*
*    prometheusK8s:*
*      retention: 15d*
*      volumeClaimTemplate:*
*        spec:*
*          resources:*
*            requests:*
*              storage: 100Gi*
EOF
configmap/cluster-monitoring-config configured

  3. Verify user workload monitoring pods are created

    $ oc get pods -n openshift-user-workload-monitoring
NAME                                   READY   STATUS    RESTARTS   AGE
prometheus-operator-xxxxx              2/2     Running   0          2m
prometheus-user-workload-0             6/6     Running   0          1m
prometheus-user-workload-1             6/6     Running   0          1m
thanos-ruler-user-workload-0           4/4     Running   0          1m
thanos-ruler-user-workload-1           4/4     Running   0          1m

    It may take 2-3 minutes for all pods to reach Running status.

2. Configure DCGM Exporter Service Monitor

The DCGM Exporter needs to be configured for Prometheus scraping.

  1. Label the nvidia-gpu-operator namespace for monitoring

    $ oc label namespace nvidia-gpu-operator openshift.io/cluster-monitoring=true
namespace/nvidia-gpu-operator labeled

    This label enables the user-workload Prometheus to discover ServiceMonitor resources in this namespace.

  2. Verify the DCGM Exporter service exists

    $ oc get svc -n nvidia-gpu-operator | grep dcgm
nvidia-dcgm-exporter       ClusterIP   172.30.xxx.xxx   <none>        9400/TCP   15m

  3. Check if ServiceMonitor was created by the GPU Operator

    $ oc get servicemonitor -n nvidia-gpu-operator
NAME                   AGE
nvidia-dcgm-exporter   15m

  4. Inspect the ServiceMonitor configuration

    $ oc get servicemonitor nvidia-dcgm-exporter -n nvidia-gpu-operator -o yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: nvidia-dcgm-exporter
  namespace: nvidia-gpu-operator
spec:
  endpoints:
  - interval: 30s
    port: metrics
    scheme: http
  selector:
    matchLabels:
      app: nvidia-dcgm-exporter

3. Verify Metric Collection

  1. Access the Prometheus web console

    $ oc get route -n openshift-monitoring
NAME                HOST/PORT
prometheus-k8s      prometheus-k8s-openshift-monitoring.apps.cluster.example.com

    Open the Prometheus route in your browser. You may need to authenticate with your OpenShift credentials.

  2. Query for DCGM metrics in the Prometheus UI

    In the Prometheus query interface, enter:

    DCGM_FI_DEV_GPU_UTIL

    You should see metrics for all GPU devices across your cluster:

    DCGM_FI_DEV_GPU_UTIL{gpu="0", UUID="GPU-xxxxx", device="nvidia0",
  exported_namespace="nvidia-gpu-operator", pod="nvidia-dcgm-exporter-xxxxx"} 45

  3. Query for GPU memory usage

    DCGM_FI_DEV_FB_USED / DCGM_FI_DEV_FB_FREE * 100

    This calculates the GPU memory utilization percentage.

  4. Verify metrics from multiple nodes

    sum by (node) (DCGM_FI_DEV_GPU_UTIL)

    This should show GPU utilization aggregated by node.

4. Test Metric Export with Sample Workload

Create a GPU workload to generate metrics.

  1. Deploy a simple CUDA workload

    $ cat <<EOF | oc apply -f -
apiVersion: v1
kind: Pod
metadata:
*  name: gpu-burn*
*  namespace: default*
spec:
*  restartPolicy: Never*
*  containers:*
*  - name: gpu-burn*
*    image: quay.io/nvidia/gpu-burn:latest*
*    args: ["60"]  # Run for 60 seconds*
*    resources:*
*      limits:*
*        nvidia.com/gpu: 1*
EOF
pod/gpu-burn created

  2. Monitor GPU utilization in real-time via Prometheus

    Query:

    DCGM_FI_DEV_GPU_UTIL{exported_namespace="default"}

    You should see GPU utilization spike to near 100% while the gpu-burn workload is running.

  3. Check GPU temperature during load

    DCGM_FI_DEV_GPU_TEMP

    Temperature should increase as the GPU processes the workload.

  4. View GPU power consumption

    DCGM_FI_DEV_POWER_USAGE

    Power draw will increase during intensive GPU operations.

5. Configure Metric Retention

For production MaaS deployments, configure appropriate metric retention.

  1. Update user workload monitoring retention

    $ cat <<EOF | oc apply -f -
apiVersion: v1
kind: ConfigMap
metadata:
*  name: user-workload-monitoring-config*
*  namespace: openshift-user-workload-monitoring*
data:
*  config.yaml: |*
*    prometheus:*
*      retention: 30d*
*      volumeClaimTemplate:*
*        spec:*
*          resources:*
*            requests:*
*              storage: 200Gi*
EOF
configmap/user-workload-monitoring-config created

    Adjust retention period and storage based on your organization’s requirements and compliance needs.

  2. Verify Prometheus restarts with new configuration

    $ oc get pods -n openshift-user-workload-monitoring -w
prometheus-user-workload-0   6/6   Terminating   0   15m
prometheus-user-workload-0   6/6   Running       0   30s

You have successfully enabled user-workload monitoring and configured GPU metric collection via DCGM Exporter. Prometheus is now continuously scraping GPU telemetry, making it available for visualization, alerting, and historical analysis.
